Changelog & Architecture Evolution
v1.0.0 LATEST
April 2026

🏗️ 3-Tier POM Architecture

  • Overhauled AI Engine to natively generate the 3-Tier Page Object Model (POM) architecture mimicking manual framework patterns.
  • Page Layer: Automatically generates pages/[safeName]Page.js extending BasePage to define locators and user workflows.
  • Verification Layer: Isolates assertions by dynamically generating verification/[safeName]Verification.js utilizing strict @playwright/test expects.
  • Test Layer: Produces lightweight, modular tests/[safeName]/Batch_[N].spec.js files that execute flows using the generated Page Objects.
  • Implemented intelligent JSON parsing in automation_controller.js to simultaneously parse and save all three files per batch generation cycle.

🚀 Playwright Migration & Native Execution

  • Fully migrated the underlying extraction engine from selenium-webdriver to native Playwright APIs.
  • Upgraded browser_service.js to leverage Playwright's blazing-fast Chromium instances for full-page screenshotting and DOM extraction.
  • Re-architected the Node.js execution loop to seamlessly run npx playwright test dynamically on newly generated scripts.
  • Resolved deep ES Module (`import`) vs CommonJS (`require`) configuration conflicts to allow perfect compilation of `BasePage` dependencies within AI-generated tests.

⚙️ Framework Centralization

  • Removed the isolated package.json from the E2E sub-directory, unifying all Playwright and AI dependencies into the root project directory.
  • Introduced a global npm run ai command for frictionless execution of the automation engine from anywhere.
  • Re-routed all AI artifacts (Layouts, Memory, Test Data, Observations) to central framework folders for a single source of truth.
